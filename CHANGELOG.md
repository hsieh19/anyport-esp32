# AnyPort ESP32 固件更新日志 (Changelog)

所有对 AnyPort ESP32 网关固件的重大更改都将记录在此文件中。
  
## [1.5.1] - 2026-04-10
### 新增 (OTA 冗余与云端治理)
- **原生双区固件安全回滚 (Dual-Bank Rollback)**：
    - 新增基于 ESP32 原生 A/B 分区的“一键回滚”功能。在遇到严重配置异常或破坏性更新时，用户可通过进入“系统固件更新模式”一键倒退回上一个稳定镜像。
    - **NVS 版本快照**：修复了 Arduino 框架下通过 `esp_app_desc_t` 读取历史固件只能获取到底层 IDF 版本的痛点。现在升级前会将 `FIRMWARE_VERSION` 主动锚定在 NVS 空间，以供回滚界面进行精确的人性化预演显示。
- **R2 云端仓库流转过期策略 (Retention Policy)**：
    - 废弃了粗暴的 Action Delete 重建机制，现在采用云原生 `aws-cli` 工具链实现细粒度的 S3 存储桶管理。
    - 在自动保障 R2 存储空间健康的策略下，默认只保留最近的 **10** 个可用固件版本。一旦超量，则依据时间戳流转销毁最古老的发布包，防范无限量堆积。

### 优化与修复 (底层可靠性突击)
- **W5500 启动链路协商极速进化方案**：
    - 全面召回并优化了曾被移除的“多轮自适应自协商探测”方案 (Round 0-3)：针对部分设计缺陷（不支持 Auto-MDIX）的 W5500 模块，在链路未响应时系统会自动介入实施“强制降级至 100M 全双工 (Forced 100M FD)”的软复位操作。
    - **极速启动**：大幅收紧了超时阈值，将无外部网线插入时的最坏自检卡住时间从 **~20 秒猛烈缩短至 ~9 秒内**，这给既不插网线又重度使用 WiFi 透传的用户带去了丝滑的加载体验。
  
## [1.5.0] - 2026-04-10
### 新增 (全链路混合网络支持)
- **WiFi 转以太网透传模式 (Mode 5)**：
    - 引入全新的“有线转无线”透明桥接模式。ESP32 可在 WiFi 端口上建立监听（TCP Server），将来自无线的请求无缝转发至连接在 W5500 以太网口的底层有线网段设备。
- **全模式混合网络引擎 (Hybrid Network)**：
    - 对底层核心逻辑进行了重构，解耦了对单一网络接口的依赖。
    - 在“从站模拟器模式”及“RTU-TCP 协议互转模式”中，新增**网络接口选择**。用户可在 `自动 (双网共存)`、`仅以太网 (W5500)`、`仅 WiFi` 中自由配置数据出口/入口。

### 优化与修复 (底层可靠性与 UI/UX)
- **W5500 硬件检测与 SPI 时序加固**：
    - 彻底优化了 W5500 的硬件探测逻辑。当未接入以太网模块时可极速跳过，不再阻塞系统起动；同时严格匹配了 RST 物理复位时序与 SPI 通信频率，提升了劣质电源环境下的握手成功率，并加入了带重试机制的静态 IP 地址双向校验。
- **Web 面板界面与体验升级**：
    - **版式渲染修正**：修复了配置页面由于 HTML 容器闭合错误导致的“串口参数配置”非法向外显示溢出的 BUG，重塑了页面干净度。
    - **词条规范化**：修改工作模式名称为“WiFi 转以太网透传”及“RTU-TCP 协议互转”，并重新排序下拉菜单。
    - **OTA 原生 Markdown 解析**：升级了 Web 控制面的 OTA 模块，在资源极其受限的前提下依靠极简内联原生正则表达式，使变更日志 (Changelog) 实现漂亮的原生 Markdown 排版显示。

## [1.4.0] - 2026-04-09
### 新增 (OTA 系统与架构重构专题)
- **生产级安全 OTA 系统上线**：
    - **三段式验证架构**：引入“版本检查 ➡️ 链接签名 ➡️ 加密下载”流程。利用 Cloudflare Workers + HMAC-SHA256 算法生成限时（10分钟）签名的 R2 下载链接，确保固件分发安全性。
    - **固件存在性预检**：点击升级前，系统会自动校验云端 R2 存储中是否存在对应的二进制文件，彻底杜绝因文件缺失导致的“空升级”重启风险。
    - **User-Agent 审计**：OTA 请求现在携带 `AnyPort-Collector/vX.X.X` 指纹，方便在服务端进行流量审计与拦截策略配置。
- **智能化更新提醒**：
    - **后台自动巡检**：设备启动 30 秒后自动执行首次巡检，随后开启每 24 小时一次的静默轮询更新逻辑。
    - **Web UI 徽章提醒**：发现新版本时，控制面板顶部标题栏会自动显示醒目的“发现新版本”徽章。
- **工作模式重构 (独立维护状态)**：
    - **系统固件更新模式 (Mode 4)**：新增独立维护模式，将升级逻辑与生产业务逻辑分离。在升级期间可关闭 MQTT、模拟器等高负载任务，提升升级稳定性。
    - **动态 UI 适配**：优化了 Web 面板的响应逻辑，在维护模式下自动隐藏以太网配置等无关卡片，界面极致精简。
- **仓库架构重大调整**：
    - **物理拆分**：将 ESP32 固件源代码从主仓库剥离至 `anyport-esp32` 独立仓库，实现固件与前端工具的专版专修。

## [1.3.1] - 2026-04-07
### 修复与增强 (Modbus 6位寻址专题)
- **Modbus 6 位寻址全面支持 (Simulator & Core)**：
    - 修复了从站模拟器模式下 `4xxxxx` 等大地址 PLC 寻址溢出的 bug，将地址存储从 `uint16_t` 升级为 `uint32_t`。
    - 解决了输入 `410001` 后重启变为 `16875` 的逻辑截断问题。
    - 完善了地址判定逻辑，支持对 `4xxxxx` (Holding)、`3xxxxx` (Input)、`1xxxxx` (Discrete) 和 `0xxxxx` (Coils) 全系列 6 位 Modicon 格式的自动识别映射。
- **模拟器 UI 交互重大改进**：
    - **双基准地址显示**：表头新增 `PLC 地址 (Base-1)` 与 `数据地址 (Base-0)` 明确标注，彻底解决跨软件调试时的“差 1 位”困惑。
    - **实时十六进制换算**：在配置地址时，系统会自动实时计算并显示其在协议报文中的十六进制物理偏移量（如 `410001` ➡️ `2710H`），方便对照低级报文日志。
    - **内置配置指南**：在寄存器列表上方增加了醒目的提示框，引导用户根据上位机（Master）的寻址习惯正确选用 Base-0 或 Base-1。
- **模拟器性能与容量提升**：
    - 将模拟器内部寄存器池（Holding/Input）从 1000 个扩容至 **10000** 个。
    - 将线圈/离散输入池（Coils/Discrete）从 1000 位扩容至 **10000** 位。
    - 加入了 NVS 存储对齐校验，防止因结构体大小变更导致的配置数据损坏。

## [1.3.0] - 2026-04-01
### 新增 (核心突破)
- **Modbus 协议互转模式 (Mode 4)**：
    - 正式引入双向协议桥接引擎，支持“以太网主站 -> 访问串口从站 (TCP Server)”与“串口主站 -> 访问网络从站 (RTU Master)”两种转换方向。
    - 支持**总线级透明传输 (Bus-level)** 与 **单个设备地址映射 (Single Device mapping)**，适配复杂的现场从站组合。
    - 提供完整的 RS485 串口参数配置（支持 7/8位数据位，各校验位及 1/2位停止位）。

### 修复与增强 (以太网稳定性专题)
- **W5500 PHY 启动策略重构**：
    - 优化了初始化物理层时序，强制先执行硬件复位再启动 SPI，消除了启动瞬间的信号乱码，根治了“连路由器能起、直连不起”的顽疾。
    - 引入**三轮自适应链路建立机制**：Round 0 自然建链 -> Round 1 硬件复位 -> Round 2 强制 100M FD (适配部分缺陷 PHY) -> Round 3 硬件模式重试。
    - 引入基于 MAC 地址的**启动抖动 (Jitter)** 逻辑，解决了两台物理设备同时启动时因状态机步调一致而导致的直连链路建立失败。
- **Modbus TCP 传输优化**：
    - 实现了 **TCP 连接复用 (Persistent Connection)**，同一目标设备不再频繁断开重连，彻底解决了 W5500 硬件 Socket 卡在 TIME_WAIT 状态导致的 `tcp_connect_failed` 报错。
    - 增加了发送前的接收缓冲区强力清理，确保高频请求下报文的纯净度。

## [1.2.2] - 2026-03-24
### 新增
- **网络诊断增强**：Ping/Probe 响应现在包含网关本机的 IP 地址 (`localIp`) 和以太网链路状态 (`link`)，支持远程查看链路是否物理断开。
- **配置恢复**：由于之前的兼容性调整导致的 MQTT 模式 W5500 目标地址 (TCP Target) 设置失效问题已修复，现已支持远程动态切换 Modbus TCP 目标设备。

### 优化
- **串口驱动稳定性**：在 Modbus 转发逻辑中引入了“链路预热”与“缓冲区强力清理”机制，解决了在某些高频采集场景下偶发的串口回显干扰。
- **并发冲突保护**：优化了 RS485 总线忙检测逻辑，防止多端指令重叠导致的总线死锁。

## [1.2.1] - 2026-03-19
### 新增
- **实时报文监听 (Sniffer)**：从站模拟器模式新增 Web 实时报文显示窗口，支持 HEX 报文流捕获、TX/RX 染色区分及 `分:秒.毫秒` 级高精度时间戳。支持“一键清空硬件缓冲区”与“前端暂停接收”功能。
- **大地址智能映射**：重构了 Modbus 地址解析引擎，支持 0-65535 全量地址自动识别。兼容“字面量绝对地址”（如 40600）与“标准偏移地址”（如 599）的自动对齐，解决了非标上位机的兼容性痛点。

### 优化
- **模拟器响应性能**：优化了在高波特率下的 Modbus 响应 Hook 逻辑，降低了监控开启时的系统延迟。
- **UI 交互**：优化了寄存器列表的刷新频率与状态显示。

## [1.2.0] - 2026-03-14
### 新增
- **USB 转 RS485 透传模式 (Mode 3)**：支持将 ESP32-C3 作为 USB-to-RS485 高速桥接器使用。针对 ESP32-C3 简约版优化，支持通过 Web 页面手动配置串口参数（波特率、数据位、校验位、停止位）。
- **mDNS 域名访问**：支持通过 `anyport.local` (默认) 或自定义域名在局域网内访问 Web 配置面板，无需记忆 IP 地址。
- **零配置优化**：默认透传参数设为 9600 8N1，符合大多数工业设备初始设置。

## [1.1.0] - 2026-03-13

### 新增
- **从站模拟器模式**：支持 0x/1x/3x/4x 寄存器模拟，包含动态值模拟逻辑（随机、递增、递减）。
- **复合 Web 面板**：支持在网关模式与模拟器模式间动态切换及配置寄存器。
- **版本可视化**：在 Web 控制面板的页面标题（Title）和主标题（H1）中增加了固件版本号显示。

### 优化
- 完善了代码审计中发现的模块化建议。
- 优化了 MQTT 和网络初始化的稳定性。

- **MQTT 网关模式**：实现 Modbus RTU (RS485) 与 Modbus TCP (W5500) 协议转发至 MQTT。
- **网络管理**：集成 WiFi STA/AP 切换，支持 W5500 以太网静态 IP 配置。
- **Web 配置**：内置基础 HTTP 管理面板，用于设置网络参数与 MQTT 账户。
